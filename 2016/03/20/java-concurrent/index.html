发表于 2016-03-20   |   分类于 Java   |  

Java的synchronized与Lock

Java的synchronized与Lock都可以实现同步的访问，两者的不同有：

  • synchronized是Java内置的关键字，Lock是一个类，不是你Java语言内置的一种特性
  • synchronized同步操作的方式比较单一，Lock可以根据自己需求进行定制，拥有更丰富的功能特性。
  • synchronized同步操作完成后，不需要用户手动去释放锁，而Lock则需要用户手动去释放，从代码上讲synchronized更易用，Lock用不好的话，容易出现死锁。
  • synchronized同步模式过于单一，性能与Lock相比较差。

一般情况下，若对性能上不是具有特别要求的话，用synchronized比较不容易出错。但在性能要求较高的场景，选择合适的Lock，会具有更好的性能。比如：在具有大量的读取文件，写入文件较少的场景中，此时多线程去操作读，并不会发生冲突，只有写时才需要进行堵塞，若使用synchronized，则每次读都会堵塞，严重影响性能；若使用ReentrantLock,在读时加上读锁，在写时加上写锁，获取到的性能与synchronized相比是成线性增长的。

synchronized用法

synchronized即可用在方法的申明上，也可用于某个代码块。

package com.marcos.learn.concurrent;

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.stream.IntStream;

/**
 * Created by marcos on 16/3/21.
 */
public class SynchronizedObject {

    Integer index = 0;

    static Object object = new Object();
    public synchronized void synchronizedExecute() {
        System.out.println("方法上同步："+index++);
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
    }

    public void codeBlockExecute(){
        synchronized (object){
            System.out.println("代码块同步"+index++);
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }


    public static void main(String[] args) {
        SynchronizedObject object = new SynchronizedObject();
        ExecutorService executorService = Executors.newFixedThreadPool(30);
        IntStream.range(0, 100).forEach((i) -> {
            executorService.submit(object::synchronizedExecute);
        });

        executorService.shutdown();

        ExecutorService executorService1 = Executors.newFixedThreadPool(30);
        SynchronizedObject object1 = new SynchronizedObject();
        IntStream.range(0, 100).forEach((i) -> executorService1.submit(object1::codeBlockExecute));
        executorService1.shutdown();
        while (true){
            try {
                Thread.sleep(1000);
            } catch (InterruptedException e) {
                e.printStackTrace();
            }
        }
    }
}

简单的Lock用法

package com.marcos.learn.concurrent;

import java.util.concurrent.ExecutorService;
import java.util.concurrent.Executors;
import java.util.concurrent.locks.ReentrantLock;
import java.util.stream.IntStream;

/**
 * Created by marcos on 16/3/21.
 */
public class LockObject {

    private ReentrantLock lock = new ReentrantLock();
    int index = 0;
    public void executeLock(){
        lock.lock();
        System.out.println("当前值为：" + (index++));
        try {
            Thread.sleep(1000);
        } catch (InterruptedException e) {
            e.printStackTrace();
        }
        lock.unlock();
    }

    public static void main(String[] args){
        LockObject object = new LockObject();
        ExecutorService executorService = Executors.newFixedThreadPool(30);
        IntStream.range(0,100).forEach((i)->executorService.submit(object::executeLock));
    }
}
